# 언리얼 공부할 때
* GAS와 같이 특정한 기능에 대한 샘플 프로젝트를 가지고 공부할 때, 어떻게 해야하는가?
    * 가장 중요한 것은 필요한 내용을 기반으로 보면서 직접 구현해보는 것
    * 기초 기능을 따라 작성해보면서 기능에 대해 충분히 이해하고, 생각나는 기능을 구현해보는 것
    * AI를 통해서 특정 기능에 대한 개념 학습과 방향성을 가이드 받는 것도 좋음
    * 주의할 점 : 공식문서를 자주 들여다 볼텐데, 교과서식 공부처럼 모두 다 이해하고 넘어가야지 하는 것은 안 좋음 -> 반드시 직접 구현해보면서 해볼 것
    * 이때, 전제는 시행착오하는 것을 전제로 구현
        * 예제를 따라 작성하든 하다보면 반드시 후반부에는 설계적 관점에서 막힐 것임
        * 이때는 다음 적용때 이 부분을 유의했다가 적용하는 것을 회고하는 식으로 마무리

# 면접 준비
## 준비 주제
* 공통적으로 왜 쓰는지에 대한 의도가 들어가는 것이 좋음

### 로우 레벨
1. 메모리
    * 원시 포인트, 스마트 포인터 차이
        * 포인터 관리 뿐만 아니라 소유에 대한 권한에 대한 것
        * 스마트 포인터 - 메모리에 대한 권한 처리 방법
    * 얕은 복사 vs 깊은 복사
2. 객체지향 설계
3. 문법 (C++)
    * 연산자 오버라이드는 왜는지? -> 코드의 일관성을 위함
    * 커스텀 타입에 연산자 오버라이드를 작성해서 원시 자료형에서 사칙연산이나 대입하듯 일관성 있도록 사용하기 위함
    * 연산자를 오버라이드함으로써 STL과 알고리즘 라이브러리와 응용해서 사용이 가능
4. 자료구조, STL Container
    * 자료구조 설명 시, 형태나 작동 외에도 시간복잡도, 공간복잡도를 언급해주는 것이 좋음
    * Vector vs List
    * 각 자료구조의 시간복잡도, 공간복잡도 비교
### 게임 프레임워크, OS
5. 엔진
    * Update 사이클
6. OS
    * Job Queue
    * 대체로 멀티스레드 + 관련 내용
### 수학, 그래픽스
7. 수학
    * 벡터와 스칼라
    * 삼각함수를 가지고 회전공식을 설명
    * 내적 활용 예시
    * 외적 활용 예시
    * 쿼터니언 vs 오일러각
    * 짐벌락 해결방법
    * 두 점간의 거리
    * 선형대수학 행렬-기하학, 삼각함수, 벡터, 스칼라
8. 그래픽스 DirectX
    * 렌더 파이프라인
    * 버텍스 셰이더, 픽셀 셰이더 차이
    * View Port
    * SwapChain : 체인의 갯수를 설정해서 더블 버퍼링으 구현하는 방법
    * 드로우콜
    * 최적화
### 엔진
9. 엔진
    * 기본 구조
        * State들의 차이 GameState vs PlayerState
        * Actor - Pawn - Character 차이
        * PlayerController의 역할
    * GC
        * GC의 생명주기가 해제되는 상황
        * UPROPERTY, UCLASS, UFUNCTION 과 같은 매크로
    * 포트폴리오 관련 질문
        * GAS
        * 네트워킹
