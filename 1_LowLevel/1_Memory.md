# 메모리
## 더블 프리, 댕글링 포인터, 메모리 누수가 생기는 상황
* 더블 플리 (double free): 메모리가 이미 해제된 포인터(메모리 주소)를 다시 delete, free를 실행할 때 발생합니다.
* 댕글링 포인터 (dangling pointer): 이미 해제되었거나 유효하지 않은 메모리 영역을 가리키는 포인터를 말합니다. 객체가 소멸한 후에도 해당 객체의 메모리 주소를 가리키는 포인터를 사용하려고 할 때 발생합니다.
* 메모리 누수 (memory leak): 동적으로 할당된 메모리가 프로그램에 의해 제대로 해제되지 않을 때 발생합니다. 프로그램이 실행되는 동안 지속적으로 불필요한 메모리를 점유하게 되며, 이는 시스템 자원의 낭비로 이어질 수 있습니다.

* 메모리 누수 방지
* C++에서 메모리 누수를 방지하기 위한 방법으로 RAII 패턴을 사용하여 동적 메모리를 관리하는 방법이 있습니다. 

### RAII (Resource Acquisition Is Initialization) 패턴
* C++에서 자주 사용되는 프로그래밍 원칙.
* 자원을 획득하는 동시에 객체를 초기화하며, 객체의 수명이 끝나면 자원을 자동으로 해제하는 방식
* 목적
    * 이 원칙의 목적은 메모리 누수, 자원 누수, 예외 안전성 등을 관리하여 안정적이고 예측 가능한 코드를 작성하는데 있습니다.
* 구체적인 구현
    * 자원을 할당하는 클래스의 생성자에서 자원을 획득한다.
    * 클래스의 소멸자에서 자원을 해제한다.
    * 이런 방식으로 객체의 수명과 자원의 수명을 연결시킨다.
* 적용 예시
    * Smart Pointer
        * unique_ptr, shared_ptr 같은 스마트 포인터 클래스는 객체가 소멸할때, 자동으로 할당된 메모리를 해제합니다.
    * 멀티스레딩에서 Lock Guard가 있습니다.
        * 생성될 때 mutex를 받아 잠그고, 소멸될 때 mutex를 해제합니다.

## 원시 포인터와 스마트 포인터 차이
- 원시 포인터는 데이터의 메모리 주소를 가리키는 기능만 수행
- 스마트 포인터는 메모리 주소를 가리키는 기능과 함께 데이터의 메모리 관리 기능을 추가됨.<br>추가적으로 메모리에 대한 사용 권한 또는 소유 권한을 나타낼 수 있음.
    - 메모리 관리적 측면에서는 더 이상 참조가 없는 경우에 메모리를 자동 정리.
    - 메모리에 소유에 대한 권한을 처리하는 방식이 크게 3가지가 있음.
        1. `shared_ptr`
        2. `weak_ptr`
        3. `unique_ptr`
- 사용하는 이유(의도) : 포인터 관리 뿐만 아니라 소유에 대한 권한에 대한 것

### 스마트 포인터 세종류들의 차이
- 스마트 포인터 - 메모리에 대한 권한 처리 방법
1. `shared_ptr`
    - 여러 곳에서 데이터의 메모리에 대한 참조가 가능
    - reference count를 기반으로 메모리 참조 횟수를 알 수 있으며, reference count가 0이 되면 메모리를 정리해줌.
        - 정확히는 strong reference count
    - shared_ptr을 쓰는 이유 : 메모리에 대한 수명 연장
        - 공유한다는 것은 한 데이터에 대해서 여러 곳에서 소유권을 주장하는 것.
        - 여러 곳에서 하나의 데이터에 대한 메모리에 공유할 때는 댕글링 포인터에 대해 주의해야함.
        - 한 곳에서 아직 사용 중인데 다른 곳에서 메모리를 해제하는 등의 문제로 방지하기 위해서
        reference count를 이용해서 메모리에 대한 수명을 유지할 수 있음.
2. `weak_ptr`
    - shared_ptr과는 반대로 소유권을 주장하지 않고 참조하고자할 때 쓰는 것이 weak_ptr.
        - shared_ptr에 대해 strong reference count를 증가시키지 않고 참조
    - 메모리 해제에 관여하지 않고 참조할 수 있음.
        - 이 때문에 weak_ptr이 가리키는 곳은 nullptr일 수 있음.
        - 그러므로 IsValid() 를 통해서 가리키는 메모리 주소가 살아있는지 확인해야 함.
    - weak_ptr을 쓰는 이유 : 
        - shared_ptr의 사용에서 순환 참조 문제가 발생할 수 있음.
        - 서로 다른 메모리가 서로를 참조하고 있기 때문에 reference count가 1로 유지되어 해제되지 않는 문제 -> 메모리 누수 문제
        - 이 문제를 weak_ptr을 적용하여 두 메모리가 약참조를 하도록하여 순환 참조를 해결할 수 있음
        - 이와 함께, 설계적인 의도에서 사용할 수도 있는데 어떤 데이터의 소유권은 다른 곳에 있고
        weak_ptr로 참조하는 객체는 단순히 의존만한다는 의도를 명시할 수 있음
3. `unique_ptr`
    - 데이터에 대해서 단 한 곳에서만 참조 가능 -> 유일한 소유권 의미
        - 데이터의 소유 주체를 명확히 할 수 있음
    - 단 한 곳에서만 소유가 가능하기 때문에 복사 연산자가 없음
        - 다른 곳으로 소유권을 넘기고 싶다면 move sematic을 사용해서 이동시킬 수 있음
        - 또는 release()를 통해서 원시 포인터를 받고 소유권을 포기하는 것이 가능
    - unique_ptr은 선언된 스코프를 벗어나면 자동으로 메모리를 정리 (RAII)
        - 클래스에 멤버변수로 unique_ptr이 있다면, 클래스가 소멸될 때 같이 정리됨
        - 함수 내에서 선언되었다면, 함수 호출이 끝나고 스택 영역에서 정리될 때 같이 정리됨

## 얕은 복사 vs 깊은 복사
- 실질적인 데이터의 복사 없이, 한 데이터의 메모리를 참조하는 것을 얕은 복사
    * 대체로 포인터를 다른 포인터에 대입할 때 발생
- 새로운 메모리 주소를 할당받아 기존의 데이터 자체를 복사하는 것을 깊은 복사
    * 일반적으로 복사 생성자를 할때 발생

## 프로그램에서 메모리 영역과 생명주기
### 코드
* 실행할 프로그램의 코드가 저장된 영역
### 데이터
* 전역변수, static 변수 등이 저장되는 영역
* 메인 함수가 호출되기 전에 데이터 영역에 할당되고, 종료되면 같이 정리됨
### 힙
* 런타임 중 동적으로 사용할 데이터가 저장되는 영역
* 동적 생성 시, 힙 영역에 데이터가 저장되고 해제도 수동으로 해줘야 함.
### 스택
* 함수 호출 시, 생성되는 지역변수와 매개변수가 저장되는 영역
* 함수 호출이 종료되면 같이 정리됨

## malloc/free와 new/delete의 차이점
* 둘 모두 동적으로 heap 공간에 메모리를 할당하기 위해서 사용하는 공통점이 있음.
### malloc/free
* C언어부터 사용해온 메모리 할당 방식
    * 라이브러리 제공 <b>함수</b>
    * #include "stlib.h" 를 해줘야 사용 가능
* `malloc` : byte 단위의 사이즈를 매개변수로 받고, 리턴 타입이 `void*` 형
    * 반환받은 걸 원하는 타입으로 형변환 해줘야 함.
    * 즉, 메모리를 동적으로 할당하는 것만이 목적이므로 생성자를 호출할 수단이 없기에 초기값을 줄 수 없다는 단점이 있음.
    * `malloc`으로 할당한 경우 `realloc`을 활용해 메모리 크기를 조정할 수 있음.
    * 이들은 오로지 POD(Plain Old Data) 타입에 대해서만 크기를 할당해줄 수 있음.
* `free` : 또한 제공 함수로, 소멸자를 부르는 것이 아님.
### new/delete
* C++ 내에 있는 키워드로 별도의 라이브러리 추가가 없음.
    * new / delete는 일종의 연산자
* `new` : 할당할 변수의 자료형을 지정하면, 반환값이 해당 타입의 포인터로 자동 반환되므로 직접 사이즈를 입력하지 않아도 됨.
    * 클래스 객체를 동적할당하게 되면 생성자를 자동으로 호출해서, 할당과 동시에 초기화가 가능
    * `new`로 할당한 경우 직접적으로 메모리 크기를 조정할 수 없음.
    * 재할당이 빈번하다면 `malloc`을 사용하는 것이 더 유리함.
* `delete` : 소멸자를 호출하여 메모리를 해제함.
    * 즉, `free`와 다르게 소멸자를 호출하는 것이 차이점

## 포인터와 배열의 차이점?
* 포인터 : 메모리의 주소를 가리키는 자료형
    * const를 붙이지 않았다면 포인터는 다른 메모리 주소를 가리킬 수 있다.
* 배열 : 동일한 자료형을 연속적으로 나열한 자료구조
    * 배열의 이름은 배열의 첫번째 원소의 메모리 주소를 가리키고 있다.
    * 단, 배열의 주소는 const 포인터이기 때문에 메모리 주소를 바꿀 수 없다.
    * 배열의 메모리 크기는 선언 시, 생성한 크기 * 자료형의 크기
