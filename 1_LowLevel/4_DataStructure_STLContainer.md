# 자료구조
## 배열 vs 리스트
## 맵과 해시
* C++에서 Map과 Hash Map은 key-value로 이루어진 pair 객체를 저장하는 연관 컨테이너입니다.
    * 연관 컨테이너 : key와 value를 통해 관계있는 값을 묶어서 저장하는 컨테이너
* Map과 Hash Map은 내부 구현 방식에서 차이가 있습니다.
### Map
* `std::map`은 균형 이진 트리(보통 레드-블랙 트리)를 기반으로 구현됩니다.
* 각 요소는 키에 의해 자동으로 오름차순 정렬되며, 중복 키를 허용하지 않습니다.
* 시간복잡도
    * 탐색 : 일반적으로 O(log N)
    * 삽입, 삭제 : 일반적으로 O(log N)
* key에 따라 자동으로 정렬된 상태를 유지하고 할때 유용합니다.
### Hash Map
* `std::unordered_map`은 해시 테이블을 기반으로 구현됩니다.
* 해시 함수를 이용하여 key에서 해시값을 얻고, 버킷에 해시값이 해당하는 공간에 value를 저장합니다.
* 해시 함수의 결과에 따라 데이터가 저장되기 때문에 순서가 불규칙합니다.
* map과 마찬가지로 key 중복을 허용하지 않습니다.
* 시간복잡도
    * 평균적인 경우
        * 탐색, 삽입, 삭제 : O(1)
    * 최악의 경우
        * 탐색, 삽입, 삭제: O(n)
        * 해시 충돌을 해결하기 위핸 체이닝 또는 주소 확장으로 인한 경우
### 둘의 사용 경우
* `std::map` : Map
    * key-value 쌍이 특정 순서로 정렬되어야할 때 사용
    * 요소의 삽입과 삭제가 빈번하게 일어나고, 이를 정렬 상태로 유지해야할 때 유용
    * 단순히 삽입과 삭제가 빈번하다면 Hash Map이 더 유리할 순 있지만, 정렬을 해야한다면 Map이 더 유용
* `std::unordered_map` : Hash Map
    * 정렬 여부 상관없이 삽입, 삭제, 탐색 작업의 속도가 더 중요한 경우 사용
    * 특히, 대량의 데이터를 다룰 때
    * 요소의 순서가 중요하지 않거나, key-value 쌍의 정렬이 불필요한 경우
